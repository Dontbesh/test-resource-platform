# 测试平台领域上下文

该上下文描述测试团队如何登记、分配和使用机器资源，并为后续资源操作、测试用例和测试任务提供统一语言。

## Language

**机器资源**：
可被测试人员占用和使用的一台物理机或虚拟机。
_避免：设备、节点、主机（除非特指虚机的宿主机）_

**关键机器**：
凭据查看和敏感操作被额外限制为仅 `ADMIN` 可执行的机器资源。
_避免：重要机器、特殊机器_

**资源编号**：
创建后不可变、用于唯一指代一台机器资源的标识；物理机使用设备 SN，虚机使用标准 UUID。
_避免：IP、MAC、数据库自增 ID、资源名称_

**资源名称**：
用于搜索和展示、允许包含版本与架构等可读信息的机器资源名称。
_避免：把资源名称当作不可变唯一标识_

**资源池**：
按照实验室、网络区域或用途组织的一组机器资源。
_避免：组织、租户、机器组_

**资源租约**：
某位用户在明确起止时间内独占一台机器资源、并由不可变租约编号唯一标识的有效使用权记录。
_避免：预约、占用记录_

**占用**：
为当前用户立即创建一条资源租约。
_避免：预约、申请_

**释放**：
在原定结束时间之前主动终止一条资源租约。
_避免：删除占用、取消预约_

**延期**：
在资源租约仍然有效时延长其结束时间。
_避免：重新占用、修改机器占用人_

**强制释放**：
由 `ADMIN` 或 `TSE` 提前终止他人的有效资源租约，用于处理冲突、超期沟通失败或资源回收。
_避免：删除租约、管理员取消预约_

**到期**：
资源租约到达约定结束时间后的状态。
_避免：释放_

**租约事件**：
记录资源租约关键状态变化的不可变历史事实，例如占用、延期、释放、到期和强制释放。
_避免：审计日志、租约状态字段_

**预约**：
为未来时间段预先申请机器资源的行为，一期不支持。
_避免：用“预约”描述立即占用_

**平台用户**：
在测试平台中拥有唯一账号、角色和操作历史的自然人。
_避免：把机器人、应用或共享账号称为用户_

**外部身份绑定**：
一个外部渠道身份与一个平台用户之间经确认建立的唯一对应关系。
_避免：机器人账号、共享操作者_

**集成客户端**：
代表一个可信外部渠道应用、但不拥有平台业务角色的非自然人调用方。
_避免：平台用户、机器人用户、服务账号角色_

**操作草案**：
LLM 根据用户自然语言生成、由不可变草案编号标识、绑定用户与来源会话且短时有效，但尚未执行的结构化平台操作建议。
_避免：已提交操作、自动执行、Agent 决策_

**快捷命令**：
无需 LLM 参与、由确定性路由直接映射到查询、表单或业务 API 的用户操作入口。
_避免：API 指令、Prompt、自然语言意图_

## Relationships

- 一个 **资源池** 包含零到多台 **机器资源**。
- 一台 **机器资源** 可以被标记为 **关键机器**。
- 一台 **机器资源** 只有一个不可变 **资源编号**，并有一个可读 **资源名称**。
- 一台 **机器资源** 同一时刻最多存在一条有效 **资源租约**。
- 一条 **资源租约** 只属于一位用户和一台 **机器资源**。
- 对 **资源租约** 的释放和延期只按其不可变租约编号执行，不按机器资源编号猜测目标租约。
- **占用** 创建资源租约，**延期** 延长有效资源租约的结束时间，**释放** 提前终止本人资源租约，**强制释放** 提前终止他人资源租约，**到期** 按结束时间终止其有效性。
- 每一次 **占用**、**延期**、**释放**、**强制释放** 和 **到期** 都应产生一条 **租约事件**。
- 一个 **平台用户** 可以拥有零到多个不同渠道的 **外部身份绑定**。
- 同一渠道中的一个外部身份只能绑定一个 **平台用户**。
- 一个 **集成客户端** 可以代表已绑定的 **平台用户** 发起请求，但业务权限始终来自该平台用户。
- 一份 **操作草案** 只有在平台用户确认后，才能转换为一次经过正常授权的业务 API 调用。
- 一份 **操作草案** 只能在创建它的同一渠道和会话中，由同一平台用户确认。
- **快捷命令** 命中后直接进入确定性流程，只有未命中任何快捷命令的输入才交给 LLM。

## Example dialogue

> **Dev：**“用户在飞书里说‘占用 machine-01 两天’，这是创建预约吗？”
> **Domain expert：**“不是。它立即创建一条两天的资源租约；预约是未来时段的能力，一期不支持。”
>
> **Dev：**“这次占用记录成飞书机器人操作可以吗？”
> **Domain expert：**“不可以。系统必须通过外部身份绑定找到真实平台用户，并以该用户的角色授权和审计。”

## Flagged ambiguities

- “预约”和“占用”曾被混用——已明确：一期只有立即 **占用**，没有未来时段 **预约**。
- “释放”和“到期”曾可能指向同一结果——已明确：**释放** 是主动提前终止，**到期** 是租约自然结束。
- “IM 调用者”可能指机器人应用或发消息的人——已明确：机器人只是渠道，业务操作者是完成 **外部身份绑定** 的真实 **平台用户**。
- “集成客户端”不是第四种业务角色——已明确：它只能证明渠道应用可信，不能获得 `ADMIN`、`TSE` 或 `TE` 权限。
- “LLM 理解了用户命令”不等于操作已获授权——已明确：LLM 只能生成 **操作草案**，用户确认后仍由业务 API 完成校验和执行。
- “明确的 API 指令”被用来指用户可点选或输入的操作——统一称为 **快捷命令**；已识别但参数不完整时展示确定性卡片或表单，不调用 LLM。
- “IM 是否可以返回凭据”存在便利与泄漏风险的取舍——已明确：允许在单聊中向当前占用者返回普通机器凭据；**关键机器** 仍仅限 `ADMIN`，且所有查看都必须审计。
- “设备 SN”不能标识虚机——已明确：物理机的 **资源编号** 使用设备 SN，虚机使用创建时生成并与虚拟化层一致的标准 UUID；版本、架构和时间只进入 **资源名称**。
