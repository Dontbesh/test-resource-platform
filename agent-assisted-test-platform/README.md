# Agent 辅助测试平台实习项目

本目录用于沉淀“Agent 辅助实现测试平台”实习项目的计划、周报和阶段评审记录。项目目标是重新建设一个可持续演进的内网测试平台，而不是继续扩展或完整复刻 `radiatest`。

## 项目定位

平台按五期演进：

1. 机器资源基础管理。
2. 虚机创建、装机、远程执行和电源控制。
3. 飞书等 IM 与 LLM 助手集成。
4. 测试用例管理。
5. 测试任务管理。

本次三个月实习必须完成一期，并在一期稳定后尝试二期最小技术闭环；三至五期作为后续路线图，不计入本次必达范围。

这里的“Agent 辅助”主要指使用编码 Agent 辅助需求梳理、旧系统分析、技术调研、编码、测试、排障和文档沉淀。平台一期不强制内置 AI 功能。

## 文档说明

- `project-plan.md`：项目评估、阶段范围、12 周计划、验收标准和后续路线。
- `CONTEXT.md`：领域术语、关系和已消除的歧义。
- `docs/adr/`：影响后续实现的关键架构决策。
- `weekly-reports/`：每周进展反馈，建议按 `week-01.md`、`week-02.md` 命名。

## 正式交付件去向

本目录只保存项目管理材料。平台源码、数据库迁移、部署配置和测试代码应放在新建的独立代码仓库中，不放回 `radiatest`，也不与其共用数据库。

## 执行原则

- 先完成可试用的一期，不在一期提前实现后续业务域。
- 技术栈由实习生调研并提交 ADR，经导师评审后确定。
- 从一期开始采用 API-first 方式，机器占用、释放等能力由 Web 与后续飞书等 IM 集成复用。
- 每周必须提交可运行、可检查的增量产物。
- Agent 生成的设计和代码必须经过人工理解、审查和测试。
- 旧系统只作为需求、数据和基础设施行为参考，不直接形成新系统运行时依赖。
